3. Implement source/install/nca_writer.cpp — NCA streaming writer with NCZ support:

   Update include/install/nca_writer.hpp with concrete signatures:
   ```cpp
   #pragma once
   #include <cstdint>
   #include <vector>
   #include <functional>
   #include "nx/content_mgmt.hpp"
   #include "util/crypto.hpp"

   namespace switchpalace::install {

   struct NczSectionEntry {
       uint64_t offset;
       uint64_t size;
       uint64_t cryptoType;
       uint64_t padding;
       uint8_t cryptoKey[16];
       uint8_t cryptoCounter[16];
   };

   class NcaWriter {
   public:
       // Write a standard NCA from source stream to NCM placeholder
       // readFunc: called with (buffer, maxSize) -> returns bytes read
       // chunkSize: streaming buffer size (1MB applet, 8MB application)
       using ReadFunc = std::function<ssize_t(uint8_t* buffer, size_t maxSize)>;

       bool writeStandardNca(
           nx::ContentManager& cm,
           const NcmPlaceHolderId& phId,
           uint64_t ncaSize,
           ReadFunc readFunc,
           util::SHA256& hasher,
           std::atomic<bool>& cancel,
           std::function<void(uint64_t bytesWritten)> onChunkWritten,
           size_t chunkSize = 0x800000 // 8MB default
       );

       // Write an NCZ (compressed NCA) — decompress with zstd, re-encrypt, write
       // First 0x4000 bytes are uncompressed NCA header — write directly
       // Then parse NCZSECTN entries, decompress zstd stream, re-encrypt per section
       bool writeNczNca(
           nx::ContentManager& cm,
           const NcmPlaceHolderId& phId,
           uint64_t ncaSize, // decompressed output size
           ReadFunc readFunc,
           const std::vector<NczSectionEntry>& sections,
           util::SHA256& hasher,
           std::atomic<bool>& cancel,
           std::function<void(uint64_t bytesWritten)> onChunkWritten,
           size_t chunkSize = 0x800000
       );
   };

   // Parse NCZ section headers from the data at offset 0x4000
   // Returns the sections and advances the read position past the section header
   bool parseNczSections(const uint8_t* data, size_t dataSize,
                         std::vector<NczSectionEntry>& outSections);

   } // namespace switchpalace::install
   ```

   Implementation details for nca_writer.cpp:
   - writeStandardNca: Allocate chunk buffer (chunkSize). Loop: readFunc into buffer, SHA256::update on the data, ncm.writePlaceholder at current offset, advance offset, call onChunkWritten. Check cancel between chunks. Return false if cancel or any write fails.
   - writeNczNca:
     a. First 0x4000 bytes: read directly, write to placeholder at offset 0, hash update
     b. Read NCZSECTN header: validate magic 0x4E43535A4543544E, read sectionCount, read section entries
     c. Initialize ZSTD_DStream via ZSTD_createDStream(), ZSTD_initDStream()
     d. Allocate input/output buffers
     e. Loop: read compressed data chunk, ZSTD_decompressStream into output buffer
     f. For each decompressed chunk, determine which section(s) it spans, apply aesCtrEncrypt with the section's key and counter (counter incremented by block position)
     g. Write re-encrypted data to placeholder, hash update, call onChunkWritten
     h. Check cancel between chunks
     i. ZSTD_freeDStream on completion
   - parseNczSections: Read at offset 0x4000: validate magic, read uint64_t sectionCount, read sectionCount * NczSectionEntry structs

4. Implement source/util/crypto.cpp:
   - SHA256 class: Wrap mbedtls_sha256_context (from switch-mbedtls or libnx built-in). init() calls mbedtls_sha256_init + mbedtls_sha256_starts. update() calls mbedtls_sha256_update. finish() calls mbedtls_sha256_finish + mbedtls_sha256_free.

1. Implement source/install/nsp.cpp — NSP (PFS0) installer:

   The NSP install pipeline follows this exact sequence from RESEARCH.md:

   **NSPInstall::prepare():**
   a. Open the NSP file from SD card (fopen on sdmc:/ path)
   b. Read the PFS0 header (first sizeof(PFS0Header) bytes to get fileCount + stringTableSize)
   c. Read full header: sizeof(PFS0Header) + fileCount * sizeof(PFS0FileEntry) + stringTableSize
   d. Call m_pfs0.parseHeader() — if invalid, return false
   e. Find the CNMT NCA by calling m_pfs0.findFileBySuffix(".cnmt.nca")
   f. Calculate total install size by summing all NCA file sizes from PFS0 entries
   g. Store m_totalSize

   **NSPInstall::checkAlreadyInstalled(dest):**
   a. Open NCM content storage for dest
   b. For each NCA in the PFS0 container, construct NcmContentId from the NCA filename (NCA filenames are hex-encoded content IDs + ".nca")
   c. Call contentManager.hasContent(contentId) — if any returns true, the title is already installed
   d. Return true if already installed (caller will block with error message per user decision: "delete existing version first")

   **NSPInstall::execute(dest, onProgress, cancel):**
   a. Create ContentManager, openStorage(dest), openMetaDatabase(dest)
   b. Call cleanupStalePlaceholders() (safety net for prior crashes)
   c. Find and process CNMT NCA first:
      - Read CNMT NCA data from PFS0 container
      - Parse CNMT to extract NcmContentInfo list (content records) and NcmContentMetaKey
      - Create placeholder for CNMT NCA, write data, register
   d. For each NCA in content records (excluding CNMT):
      - Determine if this is an NCZ (check if the PFS0 entry name ends in ".ncz")
      - Create placeholder via contentManager
      - Create NcaWriter
      - Create SHA256 hasher
      - Set up ReadFunc lambda that reads from the NSP file at the NCA's offset within the PFS0 container
      - If standard NCA: call writeStandardNca()
      - If NCZ: parse NCZ sections from the first 0x4000+ bytes, call writeNczNca()
      - After write completes, finalize hash — compare against NCA filename (content ID = SHA-256 of NCA data, lower 16 bytes)
      - Register content with contentManager
      - Check cancel token between NCAs — if cancelled, call rollback() and return false
      - Report progress via onProgress callback
   e. Find and import tickets: look for ".tik" and ".cert" files in PFS0
      - For each .tik file, find matching .cert, call ticketManager.importTicket()
   f. Commit meta database
   g. Push application record (derive base title ID from CNMT: mask lower 3 nibbles of any title ID to get base)
   h. Return true

   **NSPInstall::rollback():**
   a. Call contentManager.rollbackAll() — deletes all tracked placeholders

   **NSPInstall::getTotalSize():**
   Return m_totalSize (sum of all NCA sizes, decompressed size for NCZ)

2. Implement source/install/xci.cpp — XCI (HFS0) installer:

   **XCIInstall::prepare():**
   a. Open the XCI file from SD card
   b. Seek to XCI_ROOT_HFS0_OFFSET (0xF000)
   c. Read root HFS0 header, parse with m_rootHfs0.parseHeader()
   d. Find the "secure" partition entry in root HFS0: m_rootHfs0.findFileByName("secure")
   e. Calculate secure partition absolute offset: XCI_ROOT_HFS0_OFFSET + m_rootHfs0.getDataStartOffset() + secureEntry->offset
   f. Seek to secure partition offset, read secure HFS0 header, parse with m_secureHfs0.parseHeader()
   g. Validate by checking HFS0 magic at computed offset
   h. Calculate total install size from secure partition NCA files
   i. Store m_totalSize

   **XCIInstall::checkAlreadyInstalled(dest):**
   Same as NSP — check each NCA content ID in the secure partition against NCM storage.

   **XCIInstall::execute(dest, onProgress, cancel):**
   Same pipeline as NSP but reads NCA data from the secure HFS0 partition instead of PFS0:
   a. Create ContentManager, openStorage(dest), openMetaDatabase(dest)
   b. Clean up stale placeholders
   c. Process CNMT NCA first (find in secure partition by ".cnmt.nca" suffix)
   d. For each NCA: create placeholder, stream write (standard or NCZ), verify hash, register
   e. XCI files typically don't have separate .tik/.cert files — tickets are embedded in the NCA update partition or not needed for cartridge dumps. Skip ticket import for XCI unless .tik entries are found in the secure partition.
   f. Commit meta database, push application record
   g. Report progress, check cancel token

   **XCIInstall::rollback():**
   Same as NSP — contentManager.rollbackAll()

   **File reading for XCI NCAs:**
   The ReadFunc lambda for XCI must account for the absolute offset within the file:
   ```
   absoluteNcaOffset = securePartitionAbsoluteOffset + m_secureHfs0.getDataStartOffset() + ncaEntry.offset
   ```
   Each read seeks to absoluteNcaOffset + currentReadOffset before reading.

3. Also implement the free function `detectFormat()` in a new file or in one of the existing source files:
   ```cpp
   FileFormat detectFormat(const std::string& filename) {
       // Get lowercase extension
       auto ext = filename.substr(filename.rfind('.'));
       std::transform(ext.begin(), ext.end(), ext.begin(), ::tolower);
       if (ext == ".nsp") return FileFormat::NSP;
       if (ext == ".xci") return FileFormat::XCI;
       if (ext == ".nsz") return FileFormat::NSZ;
       if (ext == ".xcz") return FileFormat::XCZ;
       return FileFormat::Unknown;
   }
   ```
   NSZ files are NSP containers with NCZ entries inside — use NSPInstall with NCZ detection per NCA.
   XCZ files are XCI containers with NCZ entries inside — use XCIInstall with NCZ detection per NCA.
   So NSZ uses the NSP code path and XCZ uses the XCI code path — the NCZ handling is inside NcaWriter, triggered when an NCA entry is detected as compressed.
