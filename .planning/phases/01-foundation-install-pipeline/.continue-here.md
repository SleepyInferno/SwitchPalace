<current_state>
Root cause of crash identified and fix applied. Needs Docker rebuild + hardware re-test.

CRASH DIAGNOSIS:
- The borealis async task loop thread (Thread 01, 132KB stack, created by pthread_create in Threading::start())
  crashes at startup in the NRO-under-hbloader context
- Crash is Instruction Abort at ELF offset 0x1f5148 (in .rodata — non-executable data section)
- NRO load base confirmed: 0x3282800000 (verified via maxwell::CmdList::moveTo at ELF 0x13d240)
- Root cause: pthread_create(NULL, NULL, ...) does not properly initialize C++ runtime state
  (TLS, exception handling, static init context) for the new thread in the NRO context
- Non-deterministic timing: runs 1&2 crashed before 'cleanup:', run 3 crashed at 'titleBar added'
  because the background thread crashes asynchronously relative to main

FIX APPLIED:
- Added -DUSE_STD_THREAD=ON to Makefile and Dockerfile cmake invocations
- This switches borealis task loop from pthread_create to std::thread, which properly
  initializes C++ runtime state for the new thread
</current_state>
