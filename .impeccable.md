# SwitchPalace Design Context

## Design Context

### Users
Nintendo Switch owners running custom firmware (Atmosphere) who install and manage game content via homebrew tools. Technically savvy; comfortable with the Switch UI paradigm. They come to SwitchPalace to get a job done — install games fast, with full confidence that the tool is safe and honest. They are not impressed by flash; they are impressed by correctness, speed, and transparency.

### Brand Personality
**Trustworthy. Precise. Premium.**

Emotional goal: quiet confidence. After an install completes, the user should feel *of course it worked* — not relief, not excitement, just the calm certainty of a tool that does exactly what it promises.

### Aesthetic Direction
Mirror the official Nintendo Switch system UI — dark charcoal backgrounds, clean white type, generous spacing, and soft rounded containers — then elevate it with a royal sapphire accent color that signals SwitchPalace as a first-class citizen of the Switch ecosystem, not a third-party afterthought.

**Visual references:** Nintendo Switch system settings, eShop dark theme, Switch home screen sidebar. The shared traits: dark backgrounds, high-contrast white text, subtle surface layering, clean focus rings, and a single dominant accent color per screen.

**Anti-references:** Garish neon homebrew UIs, excessive gradients, aggressive red/orange accents that clash with the Nintendo red brand, cluttered layouts with dense information.

**Theme:** Dark only (matches Switch system default; borealis dark theme).

### Typography
borealis bundles its own Nintendo-adjacent sans-serif font stack. Use the framework defaults and enforce hierarchy through **weight and size only** — no decorative fonts.

| Role | Size | Weight |
|------|------|--------|
| Screen Title | 28sp | 600 |
| Section Header | 22sp | 600 |
| Body / List Item | 20sp | 400 |
| Caption / Metadata / Badge | 18sp | 400 or 600 |

Sizes are borealis framework defaults (AppletFrame title 28sp, sidebar 22sp, label 20sp, header/button 18sp). Two-weight system: 400 Regular and 600 Semibold only.

### Spacing Scale
Base unit: **4px**. All spacing must snap to this grid.
Common values: `4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`

### Corner Radius
| Element | Radius |
|---------|--------|
| Cards, panels | 8px |
| Buttons | 6px |
| Dialogs | 12px |
| Badges / chips | 4px |
| Progress bar | 4px |

### Component Patterns
- **Focus ring:** 2px solid `#FFFFFF`, 2px offset — identical to official Switch UI
- **Disabled state:** Text at `#5A5A5A`, no accent color, no focus ring on controller nav
- **Progress bar:** Filled track in `#3D9CF0`, unfilled track in `#3A3A3A`, height 8px, radius 4px
- **Success indicator:** `#30D158` checkmark inline with filename after install
- **Failure indicator:** `#FF453A` × inline with filename after install
- **NAND warning:** Full-width amber banner `#FF9F0A` bg with `#1A1A1A` text — must not be dismissible without acknowledgment
- **Applet mode badge:** Persistent corner chip, `#FFD60A` bg + `#1A1A1A` text, uppercase "APPLET"
- **Speed / ETA readout:** Monospaced caption in `#ADADAD` below the progress bar

### Design Principles

1. **Earn trust through clarity.** Every state — installing, hashing, failed, cancelled — must be unambiguous. Never leave the user uncertain about what the app is doing or what happened.

2. **Belong to the platform.** Use the Switch's own visual language as the foundation. Users should feel like they never left the system UI. Override it only where SwitchPalace needs to assert its own identity (accent color, brand mark).

3. **Hierarchy over decoration.** Achieve visual order through size, weight, and color contrast — not gradients, shadows, or ornamental elements. If a decorative choice doesn't improve comprehension, remove it.

4. **Danger is always visible.** NAND operations, exFAT warnings, hash failures, and applet-mode restrictions must surface as distinct, high-contrast UI states — never buried in a settings screen or dismissible without acknowledgment.

5. **Speed is a design value.** Transfer rate and ETA are primary UI elements during installation, not secondary metadata. The user chose SwitchPalace for speed; the UI should reflect that this is something the app is proud of.
